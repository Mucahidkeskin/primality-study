# Primality-test cost datasets

Generated by `../make_datasets.sh` using `../primality.exe` (batch CSV mode).
Re-run that script to regenerate. All cost numbers are **deterministic** given
`(n, method, k, seed)` — they are operation counts, not timings.

## CSV columns (all files)

| column | meaning |
|--------|---------|
| `label` | dataset tag supplied at input |
| `bits` | bit length of n |
| `method` | Trial division / Fermat / Solovay-Strassen / Miller-Rabin / AKS / Lucas-Lehmer |
| `verdict` | PRIME / COMPOSITE / PROBABLY_PRIME / INCONCLUSIVE / NA / SKIPPED |
| `word_muls` | **low-level cost**: single-limb (32×32→64) multiplications. Best cross-method axis. |
| `word_div_steps` | single-limb division steps |
| `mod_muls`, `mod_sqrs` | **mid-level cost**: modular multiplications / squarings |
| `mod_reductions` | big-integer reductions (divisions) performed |
| `high_label`,`high_value` | **method-specific** count (divisions / rounds / poly_mults / LL_iters) |
| `k`, `seed` | rounds and PRNG seed used (for reproducibility) |

## Files

- **fig1_scaling.csv** — probable primes, 16…2048 bits, 3 reps each, k=20, AKS disabled.
  Plot `word_muls` (log y) vs `bits` (log x), one line per method.
  Finding: Fermat/SS/MR coincide; `word_muls` ∝ bits³ (~8× per bit-doubling),
  `mod_muls+mod_sqrs` ∝ bits. Trial division is INCONCLUSIVE above ~40 bits.

- **fig2_reliability.csv** — Carmichael numbers + strong pseudoprimes, k=20.
  Finding: large-factor Carmichaels (294409, 56052361, 118901521) and the
  multi-base strong pseudoprime 3215031751 **fool Fermat (PROBABLY_PRIME)** while
  Miller-Rabin and Solovay-Strassen return COMPOSITE.

- **fig2b_fermat_seeds.csv** — same big Carmichaels, k=5, seeds 1..10, Fermat vs Miller.
  Finding: Fermat fooled 30/30; Miller-Rabin fooled 0/30.

- **fig3_confidence.csv** — one 1024-bit prime, k ∈ {5,10,20,40,80}.
  Finding: cost ∝ k; at equal k the three tests cost ~the same; but for equal
  confidence Miller (4⁻ᵏ) at k beats Solovay (2⁻ᵏ) at 2k → Miller ≈ 2× cheaper.

- **fig4_aks.csv** — small primes (5…17 bits), AKS enabled (cap 20), vs Miller-Rabin.
  Finding: AKS `poly_mults`/`word_muls` explode (≈12k → 557M from 5→17 bits) while
  Miller-Rabin `word_muls` stays ~140→620. ~10⁵–10⁶× more work at 17 bits.

- **fig5_lucas.csv** — Mersenne numbers Mₚ (prime and composite exponents), LL vs MR.
  Finding: Lucas-Lehmer is deterministic and ~40× cheaper than Miller-Rabin at
  1279 bits (2.03M vs 82.8M word_muls), with cost ∝ p (one squaring per iteration).
